# Certification fixture provenance

The encoded fixtures are decoded by certification demos before use. Encoding
keeps deterministic binary test data reviewable in source distributions.

| Fixture | Origin | Decoded SHA-256 |
| --- | --- | --- |
| `image_input.png.base64` | Deterministic 32x32 red RGB PNG generated by `support.write_red_png` | `f6a996480096b673a0b79ff45d7d2c95e3962ce3c4a5d4a3a5532736beede47d` |
| `knowledge_input.pdf.base64` | Deterministic one-page PDF generated by `support.write_minimal_pdf` | `3cf3456c1f585f203aa2c853249a04fb1c431c57053eedf80f9a5fa23e831501` |
| `voice_input.wav.gz.base64` | OpenAI `gpt-4o-mini-tts` with the `alloy` voice, phrase in `voice_phrase.txt`, PCM mono 24 kHz WAV, generated 2026-07-17 and encoded with gzip level 9 | `31623eb9d325a8f488a185ac4669fa0c46e9a3134d3fa6d7405edc3686d995a2` |
| `video_input.mp4.base64` | FFmpeg 8.1.1 color source, 0.2 seconds, 16x16 H.264 MP4 | `4c8f057507d062995d5ec00431f00f9157425e9b7de506f1405456e397170873` |

The voice fixture is only the input to the protected live gate. OpenAI STT
must transcribe it, and OpenAI TTS must generate the reply WAV. The generated
speech is transcribed again, so the committed fixture cannot substitute for
either real speech API call.
